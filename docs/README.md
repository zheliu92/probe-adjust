# Documentation

This directory contains comprehensive documentation for the probe-adjust system.

## 📁 Structure

- **`api/`** - API reference documentation
  - Endpoint documentation
  - Request/response schemas
  - Authentication guides

- **`user-guide/`** - User documentation
  - Getting started guides
  - Tutorial walkthroughs
  - Best practices

- **`development/`** - Developer documentation
  - Setup instructions
  - Contributing guidelines
  - Release processes

## 📖 Documentation Types

### API Documentation (`api/`)

Complete reference for all APIs:

- **REST API Reference** - Complete endpoint documentation
- **SDK Documentation** - Client library usage
- **Authentication** - Security and auth mechanisms
- **Examples** - Code samples and use cases

### User Guide (`user-guide/`)

End-user focused documentation:

- **Getting Started** - Quick start tutorial
- **Configuration** - System setup and configuration
- **Use Cases** - Common scenarios and solutions
- **Troubleshooting** - Common issues and fixes

### Development Documentation (`development/`)

Developer-focused guides:

- **Setup Guide** - Development environment setup
- **Contributing** - How to contribute to the project
- **Release Process** - How releases are managed
- **Code Style** - Coding standards and guidelines

## 📝 Documentation Standards

- Use clear, concise language
- Include code examples where applicable
- Keep documentation up-to-date with code changes
- Follow markdown best practices

## 🔄 Contributing to Documentation

1. Documentation follows the same PR process as code
2. Test all code examples before submitting
3. Use consistent formatting and style
4. Include screenshots for UI-related documentation
